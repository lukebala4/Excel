# EULAR 2026 Preliminary Planner - Indication Classification

## Overview

This project automates the classification of disease indications from abstract/session titles in the **EULAR 2026 Preliminary Planner** spreadsheet. The Indication column (L) uses a formula to scan each title in the Title column (G) and return the relevant disease abbreviation(s).

## How the Indication Formula Works

### Approach

The formula scans the **Title** column (G) for disease-related keywords and returns standardised abbreviation(s). It uses Excel's `TEXTJOIN` + `LET` functions to check for 34 disease categories simultaneously and can return **multiple indications** per title (e.g., "RA, ILD" or "SLE, LN").

### Row Coverage

- **Rows 3-341**: Manually entered indication values (curated by hand)
- **Rows 342+**: Automated formula applied to all rows with a title

### Formula Logic

```
=LET(r, TEXTJOIN(", ", TRUE, [34 IF/SEARCH checks]), IF(r="", "N/A", r))
```

Each check follows the pattern:
```
IF(ISNUMBER(SEARCH("keyword", G342)), "ABBREVIATION", "")
```

- `SEARCH` is **case-insensitive** and matches substrings
- `TEXTJOIN` concatenates all matches with `", "` separator, ignoring blanks
- `LET` stores the result in variable `r` to avoid computing `TEXTJOIN` twice
- If no keywords match, the formula returns `"N/A"`

## Disambiguation Rules

The formula includes several rules to avoid false positives and misclassification:

### PsA vs PsO
- "Psoriatic Arthritis" or any title with "Psoriatic" returns **PsA**
- "Psoriasis" returns **PsO** only if "Psoriatic" is NOT also in the title
- This prevents titles like "Prevention of Psoriatic Arthritis in Psoriasis patients" from double-flagging as PsA + PsO when the focus is PsA

### Lupus Family (SLE vs LN vs CLE)
- "Cutaneous Lupus Erythematosus" returns **CLE** only (not SLE)
- "Lupus Nephritis" returns **LN** only
- "Systemic Lupus Erythematosus" returns **SLE**
- Bare "Lupus" (without Erythematosus/Nephritis/Cutaneous) returns **SLE**
- Titles mentioning multiple lupus subtypes return all applicable (e.g., "Systemic Lupus Erythematosus and Lupus Nephritis" returns "SLE, LN")

### SpA Family (AxSpA vs pSpA vs AS vs SpA)
- "Axial Spondyloarthritis" or "axSpA" returns **AxSpA**
- "Peripheral Spondyloarthritis" or "pSpA" returns **pSpA**
- "Ankylosing Spondylitis" returns **AS**
- Generic "Spondyloarthritis" (without axial/peripheral/ankylosing qualifier) returns **SpA**
- The SpA abbreviation uses word-boundary checks (" SpA ", " SpA,", etc.) to avoid matching common words like "space", "Spain", or "spatial"

### Vasculitis Family (GCA vs TAK vs LVV vs AAV vs EGPA)
- Each vasculitis type is checked independently
- "Giant Cell Arteritis" returns **GCA**
- "Takayasu" returns **TAK** (also catches " TAK " abbreviation and "&TAK")
- "Large Vessel Vasculitis" returns **LVV**
- Titles can return multiple types (e.g., "GCA & TAK" returns "GCA, TAK")

## Abbreviation Pattern Design

For short abbreviations (RA, SLE, GCA, etc.), the formula uses **word-boundary patterns** to avoid false positives:

| Pattern | Purpose |
|---|---|
| `" RA "` | Space before and after (mid-sentence) |
| `" RA,"` | Followed by comma |
| `" RA:"` | Followed by colon |
| `" RA-"` | Followed by dash (e.g., "RA-ILD") |
| `"(RA)"` | In parentheses |
| `RIGHT(G,3)=" RA"` | End of title |
| `LEFT(G,3)="RA "` | Start of title (followed by space) |
| `LEFT(G,3)="RA-"` | Start of title (followed by dash) |

This prevents accidental matches on common words. For example, `" SpA"` without boundaries would match "space", "Spain", "spatial" - so SpA specifically uses `" SpA "`, `" SpA,"`, `" SpA:"`, `" SpA."`, `"(SpA)"`, and `RIGHT(G,4)=" SpA"`.

The formula also uses `LEFT()` checks for abbreviations that may appear at the start of a title (RA, PMR, APS) and `RIGHT()` checks for abbreviations at the end of a title (RA, GCA, ILD, APS, SpA, SLE, PsA, TAK).

## Known Limitations

1. **Keyword-dependent**: If a title doesn't mention the disease by name or abbreviation, the formula returns N/A. Some manually-classified rows (3-341) have indications assigned based on session context rather than title keywords.

2. **No context awareness**: The formula can't distinguish primary vs secondary disease mentions. A title like "Uveitis rates in Psoriatic Arthritis patients" returns "PsA, Uveitis" even if uveitis is a secondary outcome measure.

3. **Manual rows may differ**: The manually-entered values in rows 3-341 may use slightly different conventions or clinical judgment that keyword matching can't replicate.

## Requirements

- **Excel 365** or **Excel 2021+** (the formula uses `LET` and `TEXTJOIN` which are not available in older versions)

## How to Use the Formula on New Data

1. Open `Indication_Formula.md` for the copyable formula
2. Paste the formula into cell **L342** (or the first row needing automation)
3. Drag down to fill all rows
4. To change the starting row, find-and-replace `G342` with `G[your row number]`
